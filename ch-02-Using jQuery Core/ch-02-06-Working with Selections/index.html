Some jQuery methods can be used to either assign or to read some value on a selection.

When the method is called with a value as an argument, it's referred to as a setter because it sets (or assigns) that value 

        
            $("h1").html("NOTE: the headings (h3) are written using jQuery and not in index.html, also, this text is also written by jQuery");
        // The .html() method sets all the h1 elements' html to be "NOTE: the headings (h3) are written using jQuery and not in index.html, also, this text is also written by jQuery";

When the method is called with no argument, it gets (or reads) the value of the element and is therefore called getter 

        
            // The .html() method returns the html of the first h3 element:
            let h3 = $("h3").html();
            console.log(h3); //Getters & Setters

NOTE: Setters affect all elements in a selection, whereas, getters return the requested value only for the first element in the selection, with the exception of .text(), which retrieves the values of all the elements. 

        
            let p = $("p").text();
            console.log(p); //=>this will give all the text of p tags

If you call a method on a selection and that method returns a jQuery object, you can continue to call jQuery methods on the object without pausing for a semicolon. This practice is referred to as "chaining": 

        
            $("body").find("h3").eq(1).html("Chaining");

NOTE:Setters return a jQuery object, allowing you to continue calling jQuery methods on your selection i.e., can chain the methods for setters. BUT, Getters return whatever they were asked to get, so you can't continue to call jQuery methods on the value returned by the getter, i.e., can't chain the methods for getters. 

        
            i.e., 
            // Attempting to call a jQuery method after calling a getter.
            $("h1").html().addClass("test"); // Attempting to call a jQuery method after calling a getter. i.e., chaining does not work for getters => gives us ERROR

jQuery also provides the .end() method to get back to the original selection should you change the selection in the middle of a chain: 

        
            $("body")
                .find("h3")
                .eq(2)
                .html(".end() method")
                .end() // Restores the selection to all h3s in body
                .eq(0)
                .html("Getters & Setters");
