There's often data about an element you want to store with the element. In plain JavaScript, you might do this by adding a property to the DOM element, but you'd have to deal with memory leaks in some browsers. jQuery offers a straightforward way to store data related to an element, and it manages the memory issues for you.

.data() will be used

The .data() method in jQuery is used to store and retrieve data associated with DOM elements.

Store data:

        
          $(selector).data(key, value);

Retrieve data:

        
          $(selector).data(key);

Retrieve all data:

        
          $(selector).data();

The .data() method can be particularly useful for storing extra information that needs to be available for scripting purposes but shouldn't be displayed directly in the DOM.

Practical Use Case

Imagine you have a list of items, and you want to attach additional data to each item, like an ID or other meta-information:

  • Item 1
  • Item 2
  • Item 3
        
          $("#item1").data("info", { id: 101, name: "Item One" });
          $("#item2").data("info", { id: 102, name: "Item Two" });
          $("#item3").data("info", { id: 103, name: "Item Three" });

          $("li").each(function () {
            var info = $(this).data("info");
            console.log("ID: " + info.id + ", Name: " + info.name);

            var allData = $(this).data();
            console.log("All data for " + $(this).text() + ":", allData);
          });

we can also add single values without using object as:

        
          $("#item1").data("category", "Category A");
          $("#item2").data("category", "Category A");
          $("#item3").data("category", "Category A");

          $("li").each(function () {
            var info = $(this).data("info");
            console.log("ID: " + info.id + ", Name: " + info.name);

            var allData = $(this).data();
            console.log("All data for " + $(this).text() + ":", allData);

            var category = $(this).data("category");
